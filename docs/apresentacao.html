Desafio central: desbalanceamento extremo (92,5% em uma classe) + textos jurídicos longos + poucos exemplos rotulados nas classes minoritárias (~332 no total). A métrica F1-macro é obrigatória — acurácia seria enganosa: prever sempre "Irregular" daria 92,5% de acurácia sem aprender nada. (Devlin et al., 2019)

Extração de Label

Problema: pós-2020, SITUACAO = "OFICIALIZADO" (status de publicação, não o julgamento).
Solução: extração hierárquica via regex.

Pipeline de extração de label (3 etapas):
① SITUACAO → mapeamento direto ("IRREGULAR", "REGULAR COM RESSALVA")
② SUMARIO → regex: \bcontas?\s+irregulares?\b  |  \bcontas?\s+regulares?\s+com\s+ressalva\b
③ ACORDAO → regex: \bjulgar?\s+irregulares?\b  |  \bjulgar?\s+regulares?\b

📐 Modelo 1 — TF-IDF + Regressão Logística

TF-IDF (Pedregosa et al., 2011)

Cada documento é representado como um vetor esparso de frequências de n-gramas ponderadas pela raridade do termo no corpus:

TF-IDF(t,d) = tf(t,d) × log(N / df(t))
  • max_features = 50.000
  • ngram_range = (1, 2) — unigramas + bigramas
  • sublinear_tf = True — suaviza frequências altas
  • min_df = 2 — remove termos raríssimos

Regressão Logística

Classificador linear com regularização L2. Para 3 classes usa esquema one-vs-rest:

  • C = 1,0 — força de regularização
  • solver = lbfgs — otimizador quasi-Newton
  • class_weight = "balanced" — pesos inversamente proporcionais à frequência
  • max_iter = 1.000
Vantagem: capta expressões formulaicas do SUMARIO como "CONTAS IRREGULARES" com bigramas de alta frequência → discriminação quase perfeita no campo certo.

🧠 Modelo 2 — LegalBert-pt + LoRA

⚖️

LegalBert-pt (Domingues, 2022)

BERT pré-treinado em corpus jurídico brasileiro: decisões do STF, petições, acórdãos. 126M parâmetros, vocabulário jurídico especializado.

Codifica o texto como vetores densos contextuais — cada token recebe representação influenciada por todos os outros (atenção bidireccional).

🔧

LoRA (Hu et al., 2022)

Congela os 126M parâmetros originais. Insere matrizes de baixo rank nas camadas de atenção:

W' = W₀ + BA  |  B∈ℝ^(d×r), A∈ℝ^(r×k)

r=16, α=32 nas matrizes Q e V → 1,2M parâmetros treináveis (0,93% do total). Viável em GPU T4 16 GB.

⚠️

Correção Técnica

bert.pooler.dense ausente no checkpoint. Inicialização Kaiming → pesos grandes → saturação do tanh → gradiente ≈ 0 → travamento.

Solução: reinicialização Normal(0, 0,02) + descongelamento do pooler após aplicação do LoRA.

📏 Protocolo de Avaliação

Validação Cruzada 5-Fold Estratificada

O conjunto de treino (3.110 acórdãos) é dividido em 5 partes iguais, mantendo a proporção de classes em cada parte. A cada iteração, 4 partes treinam e 1 valida:

Fold 1: [V][T][T][T][T] → F1₁
Fold 2: [T][V][T][T][T] → F1₂
Fold 3: [T][T][V][T][T] → F1₃
Fold 4: [T][T][T][V][T] → F1₄
Fold 5: [T][T][T][T][V] → F1₅
F1-macro = média(F1₁…F1₅)

RANDOM_STATE = 42 garante reprodutibilidade.

Intervalo de Confiança 95% — t-Student

Com apenas 5 folds, usamos a distribuição t com n−1 = 4 graus de liberdade (mais conservadora que a normal):

SE = σ / √n

IC₉₅% = x̄ ± t₀.₀₂₅,₄ × SE

t₀.₀₂₅,₄ = 2,776

F1-macro

Média não ponderada do F1 de cada classe — penaliza igualmente desempenho ruim nas classes minoritárias, independente do tamanho delas.

F1ₖ = 2·Pₖ·Rₖ/(Pₖ+Rₖ)    F1-macro = (F1₀+F1₁+F1₂)/3

🔍 Por que TF-IDF + SUMARIO venceu?

📝

SUMARIO é formulaico

A ementa do TCU usa linguagem padronizada. Bigramas como "CONTAS IRREGULARES" aparecem sistematicamente e são capturados com precisão pelo TF-IDF. O Transformer não ganha nada com semântica aqui — o padrão lexical é suficiente.

📉

Poucos exemplos minoritários

Por fold de treino: ~170 exemplos de Regular c/ Ressalva e ~115 de Regular. O gradiente dessas classes é insuficiente para atualizar os adaptadores LoRA de forma significativa — o modelo converge para prever quase sempre a classe majoritária.

📊

Acurácia ≠ aprendizado

Todos os modelos têm acurácia >91% — mas prever sempre "Irregular" daria 92,5% automaticamente. Por isso adotamos F1-macro: modelo trivial teria apenas F1 ≈ 0,32. (Medvedeva et al., 2020)

Achado principal — consistente com a literatura de baixo recurso: em datasets jurídicos com <5.000 exemplos e desbalanceamento >90%, TF-IDF com n-gramas supera Transformers com LoRA em +33 p.p. de F1-macro.

🎯 Threshold Ótimo para Controle Externo

Função de Custo Assimétrica

No controle externo, o custo de não detectar uma irregularidade (FN) é muito maior que o custo de um falso alarme (FP):

Custo = 10 × FN + 1 × FP

Varredura em [0,05; 0,95] com passo 0,005. Threshold padrão = 0,50 → ótimo encontrado = 0,08

Com threshold 0,08: qualquer acórdão onde P(Irregular) ≥ 8% é sinalizado como Irregular. Política ultra-conservadora — zero falsos negativos.

🏛️ Impacto para o Serviço Público

🔎

Radar de Risco

Classifica acórdãos em tempo real, priorizando os de maior risco para investigação imediata. Auditores focam energia onde o risco é maior — não em triagem manual.

📡

Escala sem Ampliar Quadro

O modelo processa em segundos o que levaria dias de análise manual, viabilizando cobertura de 100% dos acórdãos de Saúde e Educação sem ampliar o quadro de servidores.

💡

Explicabilidade LIME

(Ribeiro, Singh & Guestrin, 2016) — identifica quais termos sustentam cada predição. Essencial para adoção governamental: decisões algorítmicas precisam ser auditáveis e transparentes.

Código e Dados Abertos

✅ Conclusão e Trabalho Futuro

Conclusão

  • Melhor modelo: TF-IDF + SUMARIO — F1-macro = 0,7477 [IC95%: 0,687; 0,809]
  • Em cenário de baixo recurso + desbalanceamento extremo, TF-IDF supera LegalBert-pt em 33 p.p.
  • SUMARIO é formulaico → bigramas capturam "CONTAS IRREGULARES" com alta precisão
  • LegalBert-pt limitado por ~185 exemplos minoritários por fold — sinal de gradiente insuficiente
  • Threshold = 0,08 → Recall-Irregular = 1,000 — zero irregularidades não detectadas

Trabalho Futuro
